# Contributing to HEXSONIC

## Scope

HEXSONIC is focused on self-hosted AI/original music workflows. Contributions should align with that direction.

## Before You Start

- Open an issue for large changes.
- Keep changes focused and reviewable.
- Avoid unrelated refactors in the same PR.

## Development Setup

```bash
./setup.sh
make build
make test
make lint
```

## Pull Request Guidelines

- Describe the problem and the solution clearly.
- Include testing notes and any migration impact.
- Update `README.md` if behavior/setup changes.
- Do not include secrets, private keys, or real credentials.

## Commit Quality

- Prefer small, coherent commits.
- Use clear commit messages (what changed and why).

## Legal/Content Note

Features intended for copyright infringement or unauthorized distribution will not be accepted.
